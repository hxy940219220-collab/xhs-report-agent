# Security Policy

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for this repository. Do not include API keys, account cookies, report contents, or other sensitive data in a public Issue.

Include the affected version, reproduction steps, impact, and the smallest safe proof of concept. Reports involving credential storage, the publishing confirmation boundary, duplicate-publish protection, or browser-session isolation will be prioritized.

## Supported version

Security fixes target the latest commit on the default branch while the project remains in Beta.
